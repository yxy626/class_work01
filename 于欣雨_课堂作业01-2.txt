  关系型数据库是建立在关系模型上基础的数据库，简单来说，关系模型可以看做是一个二维表，那么关系型数据库就是由多张能互相联接的二维表格组成的数据库。
  目前世界上主流的关系型数据库有：Oracle、DB2、SQL Server、Sybase ASE和MySQL等。
  先说说Oracle，全名Oracle Database，是甲骨文公司研发出来的一套关系型数据库，它是在数据库领域一直处于领先地位的产品。从开放性角度来说，Oracle支持所有主流平台上运行，完全支持所有的工业标准。采用完全开放策略，支持多用户，大事务量的事务处理，Oracle开放性很好。从可伸缩性和并行性来看，Oracle的并行服务器对各种UNIX平台的集群机制都有着相当高的集成度。安全性上，Oracle获得最高认证级别的ISO标准认证。性能上，它也是目前世界上性能最好的关系型数据库，它保持开放平台下的TPC-D和TPC-C的世界记录。操作上，较复杂，同时提供GUI和命令行，在windowsNT和unix下操作相同。向下兼容性好，风险几乎是无。可以说Oracle数据库系统是目前世界上流行的关系数据库管理系统。
  DB2是美国IBM公司开发的一套关系型数据库管理系统，主要应用于大型应用系统，具有较好的可伸缩性，可支持从大型机到单用户环境，应用于所有常见的服务器操作系统平台下。能在所有主流平台上运行（包括windows），最适于海量数据。DB2在企业级的应用最为广泛，在全球的500家最大的企业中,几乎85%以上用DB2数据库服务器。它具有很好的并行性。DB2把数据库管理扩充到了并行的、多节点的环境。数据库分区是数据库的一部分，包含自己的数据、索引、配置文件、和事务日志。数据库分区有时被称为节点安全性。安全性上来说，它也获得了最高认证级别的ISO标准认证。它的性能较高适用于数据仓库和在线事物处理。操作方面来说，操作简单，同时提供GUI和命令行，在windowsNT和unix下操作相同。
  SQL Server它最初是由Microsoft、Sybase 和Ashton-Tate三家公司共同开发的,在Windows NT 推出后，Microsoft与Sybase 在SQL Server 的开发上就分道扬镳了，Microsoft 将SQL Server移植到Windows NT系统上，专注于开发推广SQL Server 的Windows NT 版本。Sybase 则较专注于SQL Server在UNⅨ操作系统上的应用，即上方提到的Sybase ASE。作为Microsoft旗下的一员，目标是将用户和Windows系统绑定，所以只能在windows系统上使用，开放性较差。而且windows平台的可靠性，安全性和伸缩性是非常有限的。它不象unix那样久经考验，尤其是在处理大数据库。安全性也较差，至今未获得安全认证证书。性能方面，遇到多用户时会出现一些问题。操作简单，但只有图形界面。
  Sybase ASE是美国Sybase公司研制的一种关系型数据库系统，是一种典型的UNIX或WindowsNT平台上客户机/服务器环境下的大型数据库系统。能在主流平台上使用，但在多平台的混合环境中，会有一些问题。它也获得最高认证级别的ISO标准认证。性能接近于SQL Server，但在UNIX平台下的并发性要优与 SQL Server。操作上较复杂，同时提供GUI和命令行。但GUI较差，常常无法及时状态。
  MySQL是一个小型关系型数据库管理系统，开发者为瑞典MySQL AB公司。在2008年1月16号被Sun公司收购。目前MySQL被广泛地应用在Internet上的中小型网站中。由于其体积小、速度快、总体拥有成本低，尤其是开放源码这一特点，许多中小型网站为了降低网站总体拥有成本而选择了MySQL作为网站数据库。MySQL能在所有主流平台上运行。MySQL为自由软件，没有获得任何安全认证。同时，它支持多用户操作，性能较高。 MySQL有图形界面，操作简单。上个学期在学搜智时，便接触使用了MySQL，照着老师的教学视频，也可以做一个简单的术语库，操作确实很简单。
  通过对比，Oracle和DB2使用较广较多，尤其是在一些大型企业里。SQL Server使用比较局限。另外，通过搜索，除了MySQL，其他数据库应该都是收费的，性能也很好，所以MySQL很适合用来教学和中小型企业使用（不知道这个推测对不对）。
